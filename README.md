# 🧠 Stock-Genius-System

Stock-Genius-System 是一套  
**以「市場風險控管」為核心的自動化量化監控系統**。

本專案專注於：
> 「在極端市場風險（黑天鵝）出現時，  
> 系統是否應該停止進攻？」

---

## 📊 系統設計理念

市場中最大的風險，往往不是模型失準，
而是 **在不該進攻的時候仍然進攻**。

因此本系統以「風險狀態」為第一優先，
透過新聞事件與歷史行為，自動調整 AI 系統的行為模式。

---

## 🚦 風險分級（Risk Levels）

| 等級 | 說明 |
|----|----|
| L1 | 一般市場新聞 |
| L2 | 中度風險事件 |
| L3 | 高風險警示（降速、防守） |
| L4 | 系統性風險（黑天鵝，全面防守） |

當 L4 觸發時：
- AI 進攻行為停止
- 系統進入防守 / 觀察模式
- 自動記錄風險期間行為
- L4 結束後產生 AI 表現回顧報告

---

## 🧠 核心模組

- `news_radar.py`  
  市場新聞雷達與黑天鵝分級引擎

- `ai_tw_post.py / ai_us_post.py`  
  台股 / 美股 AI 分析（具 L4 自動中斷）

- `l4_ai_performance_report.py`  
  L4 事件結束後 AI 表現回顧與統計

- `l4_postmortem_report.py`  
  黑天鵝事件事後分析（Postmortem）

---

## 🤖 自動化架構

- GitHub Actions：排程與風險狀態控制
- Discord Webhook：即時風險通報
- CSV / Flag 機制：系統狀態持久化
